## 📥 用户需要做的

### 必需操作：下载并放置字体文件

#### 1. 下载字体
- **字体名称**: Pixelify Sans
- **下载地址**: https://fonts.google.com/specimen/Pixelify+Sans
- 点击 "Download family" 按钮

#### 2. 解压并找到文件
- 解压 ZIP 文件
- 进入 `static` 文件夹
- 找到 `PixelifySans-Regular.ttf`

#### 3. 放入项目
- 复制 `PixelifySans-Regular.ttf`
- 粘贴到项目的 `/public/fonts/` 文件夹

#### 4. 重启服务器
```bash
npm run dev
```

## 🔄 字体加载机制

### 当前方案

```
浏览器加载页面
  ↓
读取 CSS 的 @font-face 配置
  ↓
尝试加载 /public/fonts/PixelifySans-Regular.ttf
  ↓
如果文件存在
  → ✅ 显示像素字体
  ↓
如果文件不存在
  → ⚠️ 显示系统默认 sans-serif 字体
```

### 优势

✅ **完全离线可用** - 不需要任何网络连接  
✅ **加载速度最快** - 本地文件，< 50ms  
✅ **隐私保护** - 不请求第三方服务  
✅ **稳定可靠** - 不依赖外部 CDN  
✅ **简单清晰** - 单一字体来源

## 🆘 故障排查

### 字体不显示？

#### 检查清单
- [ ] 文件名是否为 `PixelifySans-Regular.ttf`（区分大小写）
- [ ] 文件是否在 `/public/fonts/` 文件夹中
- [ ] 是否重启了开发服务器
- [ ] 是否清除了浏览器缓存

#### 调试步骤

1. **检查文件路径**
   ```
   VSCode 左侧文件浏览器
   → public
     → fonts
       → PixelifySans-Regular.ttf ← 应该在这里
   ```

2. **检查浏览器控制台**
   - 按 `F12` 打开开发者工具
   - 切换到 "Console" 标签
   - 查看是否有字体加载错误

3. **检查网络请求**
   - 按 `F12` 打开开发者工具
   - 切换到 "Network" 标签
   - 刷新页面
   - 搜索 "PixelifySans"
   - 查看状态码（应该是 200）

4. **重启服务器**
   ```bash
   # 在 VSCode 终端
   Ctrl+C  # 停止
   npm run dev  # 重启
   ```

5. **清除浏览器缓存**
   ```
   Ctrl+Shift+Delete
   → 勾选"缓存的图片和文件"
   → 清除数据
   ```
